`SESSION_STATE.schema.json` defines the structure of the working session state file stored at `/mnt/data/_SESSION_STATE.json`. This page documents every field, its type, constraints, and purpose. The file is validated against `schemas/SESSION_STATE.schema.json` using JSON Schema Draft 2020-12.

## Required fields

<ParamField path="protocol" type="string" required>
  Constant value: `"canon-boundary-guard:gpt-project-adapter"`. Identifies the protocol family. Any other value causes schema validation to fail.
</ParamField>

<ParamField path="protocol_version" type="string" required>
  Constant value: `"1.1"`. Identifies the schema revision. Must match exactly.
</ParamField>

<ParamField path="state_seq" type="integer" required>
  Non-negative integer (`minimum: 0`). Starts at `0` after bootstrap and increments by 1 with each Mode B or Mode C state-changing decision. The value must match `seq` in any associated `CANON_STATE_DELTA`.
</ParamField>

<ParamField path="updated_at" type="string" required>
  ISO-8601 timestamp recording when the state was last written. Example: `"2025-07-14T09:15:00+00:00"`.
</ParamField>

<ParamField path="active_l0_sources" type="array" required>
  Array of L0 source objects representing every source surface that has been inspected in this session. Must not be empty after bootstrap — see [First-install note](#first-install-note). Each item conforms to the [active\_l0\_sources items](#active_l0_sources-items) schema.
</ParamField>

<ParamField path="authorized_deltas" type="array" required>
  Array of authorized delta objects tracking every Mode B/C decision that has been approved. May be empty. Each item conforms to the [authorized\_deltas items](#authorized_deltas-items) schema.
</ParamField>

<ParamField path="open_conflicts" type="array" required>
  Array of conflict objects for any provenance conflicts that have been detected. May be empty. Each item conforms to the [open\_conflicts items](#open_conflicts-items) schema.
</ParamField>

<ParamField path="pending_decisions" type="array" required>
  Array of decision objects for questions awaiting operator resolution. May be empty. Each item conforms to the [pending\_decisions items](#pending_decisions-items) schema.
</ParamField>

<ParamField path="last_persistent_artifacts" type="array" required>
  Array of artifact objects recording every durable artifact produced since the last state reset. May be empty. Each item conforms to the [last\_persistent\_artifacts items](#last_persistent_artifacts-items) schema.
</ParamField>

## Optional field

<ParamField path="last_state_hash" type="string">
  SHA-256 hash of the previous state, in the format `"sha256:<hex>"`. Used by `validate_state.py` and `artifact_fingerprint.py` to verify continuity between state files. Omit if no prior hash is available.
</ParamField>

## active\_l0\_sources items

Each object in `active_l0_sources` records one inspected source surface. `additionalProperties` is `false` — no extra keys are permitted.

<ParamField path="source_id" type="string" required>
  A path, file ID, or human-readable label that uniquely identifies the source within this session. Examples: `"canon-boundary-guard-gpt/references/protocol.md"`, `"uploaded:state_2025-07-14.json"`.
</ParamField>

<ParamField path="source_type" type="string" required>
  Enum describing the nature of the source:

  | Value | Meaning |
  |---|---|
  | `project_file` | A file present in the ChatGPT Project |
  | `project_source` | A source saved as a Project Source |
  | `uploaded_file` | A file uploaded directly into the conversation |
  | `local_file` | A file read from the local filesystem via tool |
  | `tool_output` | Output produced by a tool invocation |
  | `command_output` | Output produced by a shell command |
  | `verified_external_source` | An external URL or resource verified by the operator |
  | `state_file` | A `_SESSION_STATE.json` or state-equivalent file |
</ParamField>

<ParamField path="status" type="string" required>
  Enum describing the current accessibility of the source:

  | Value | Meaning |
  |---|---|
  | `inspected` | Source was read and verified this session |
  | `unavailable` | Source was expected but could not be accessed |
  | `conflicted` | Source was found but conflicts with another registered source |
</ParamField>

<ParamField path="notes" type="string">
  Optional free-text annotation. Use to record inspection details, anomalies, or cross-references to `open_conflicts` entries.
</ParamField>

## authorized\_deltas items

Each object in `authorized_deltas` records one approved Mode B or Mode C decision. `additionalProperties` is `false`.

<ParamField path="delta_id" type="string" required>
  Unique identifier for this delta. Typically matches the `seq` counter or a human-readable label, e.g. `"delta-3"`.
</ParamField>

<ParamField path="authorized_at" type="string" required>
  ISO-8601 timestamp when the operator authorized this delta.
</ParamField>

<ParamField path="scope" type="string" required>
  Human-readable description of what this delta covers. Example: `"Add chapter-2 draft as project source"`.
</ParamField>

<ParamField path="status" type="string" required>
  Enum tracking persistence lifecycle:

  | Value | Meaning |
  |---|---|
  | `pending_persistence` | Delta authorized but the artifact has not yet been durably saved |
  | `persisted` | Artifact has been saved and confirmed by the operator |
  | `revoked` | Authorization was withdrawn before or after persistence |
</ParamField>

<ParamField path="target" type="string">
  Optional path or label identifying the artifact that this delta targets. Example: `"/mnt/data/chapter-2-draft.md"`.
</ParamField>

## open\_conflicts items

Each object in `open_conflicts` records a detected provenance conflict. `additionalProperties` is `false`.

<ParamField path="conflict_id" type="string" required>
  Unique identifier for this conflict. Example: `"conflict-1"`.
</ParamField>

<ParamField path="description" type="string" required>
  Human-readable explanation of what conflict was detected and which sources are involved.
</ParamField>

<ParamField path="status" type="string" required>
  Enum tracking resolution lifecycle:

  | Value | Meaning |
  |---|---|
  | `open` | Conflict has been detected and is not yet resolved |
  | `resolved` | Operator provided a resolution; conflict is closed |
  | `deferred` | Resolution explicitly postponed to a future session |
</ParamField>

## pending\_decisions items

Each object in `pending_decisions` records a question that must be resolved by the operator before work can continue. `additionalProperties` is `false`.

<ParamField path="decision_id" type="string" required>
  Unique identifier for this pending decision. Example: `"decision-2"`.
</ParamField>

<ParamField path="question" type="string" required>
  The exact question the operator must answer. Should be precise enough that the answer can unambiguously close the decision.
</ParamField>

<ParamField path="blocking" type="boolean">
  When `true`, this decision blocks all Mode B/C operations until resolved. When `false` or absent, work may continue under constraints.
</ParamField>

## last\_persistent\_artifacts items

Each object in `last_persistent_artifacts` records one durable artifact produced since the last state reset. `additionalProperties` is `false`.

<ParamField path="artifact_id" type="string" required>
  Unique identifier for this artifact. Example: `"artifact-1"`.
</ParamField>

<ParamField path="artifact_type" type="string" required>
  Enum classifying the artifact:

  | Value | Meaning |
  |---|---|
  | `project_source` | Saved as a ChatGPT Project Source |
  | `downloaded_file` | Downloaded by the operator from the conversation |
  | `canvas` | A Canvas document |
  | `document` | A generic document artifact |
  | `state_file` | A SESSION_STATE or CANON_STATE_DELTA file |
  | `other` | Any artifact not covered by the above types |
</ParamField>

<ParamField path="created_at" type="string" required>
  ISO-8601 timestamp when the artifact was created or marked as persisted.
</ParamField>

<ParamField path="path_or_label" type="string">
  Optional path or human-readable label identifying where the artifact is stored. Example: `"/mnt/data/_SESSION_STATE.json"` or `"chapter-2-draft (Project Source)"`.
</ParamField>

## First-install note

<Warning>
  The schema-minimal shape shown in `state-and-recovery.md` — with an empty `active_l0_sources` array — is **not** a valid first-install template. It is schema-valid (the array is allowed to be empty by the schema), but it does not satisfy the protocol's bootstrap requirement.

  After bootstrap, `active_l0_sources` **must** contain entries for all inspected bootstrap surfaces, at minimum:

  - `canon-boundary-guard-gpt/SKILL.md`
  - `canon-boundary-guard-gpt/references/protocol.md`
  - `canon-boundary-guard-gpt/references/gpt-project-adapter.md`
  - `canon-boundary-guard-gpt/references/state-and-recovery.md`
  - `canon-boundary-guard-gpt/schemas/SESSION_STATE.schema.json`

  A state file with an empty `active_l0_sources` submitted as first-install state will be rejected by a conformant adapter.
</Warning>
